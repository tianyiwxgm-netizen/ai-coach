# 方法论决策树

> 遇到问题不知道用什么方法？按这棵树走。

---

## 1. 总决策树

```
你要用AI做什么？
│
├─ 单次问答 ──────────────────────────── 直接提问（零样本）
│   └─ 回答质量不够？ ──────────────── 加上下文 / 少样本示例
│
├─ 需要外部知识 ─────────────────────── 搜索增强 / RAG
│   ├─ 实时信息？ ──────────────────── 联网搜索 / MCP工具
│   └─ 私有知识库？ ────────────────── RAG系统
│
├─ 多步骤任务 ──────────────────────── 链式提示 / 工作流
│   ├─ 步骤固定？ ──────────────────── Prompt Chaining（编排式）
│   ├─ 需要判断分支？ ──────────────── Router + 条件分支
│   └─ 需要自主决策？ ──────────────── Agent（ReAct / Tool Use）
│
├─ 生产部署 ────────────────────────── 评估 + 护栏 + 可观测
│   ├─ 单一功能？ ──────────────────── 结构化输出 + 护栏
│   └─ 复杂系统？ ──────────────────── 多Agent + 监督者
│
└─ AI编程 ──────────────────────────── Claude Code / Cursor
    ├─ 新功能开发？ ────────────────── 上下文工程 + 迭代对话
    ├─ 调试修复？ ──────────────────── 提供错误信息 + 代码上下文
    └─ 架构设计？ ──────────────────── 推理模型 + 思维链
```

## 2. 按任务类型速查表

| 任务类型 | 首选方法 | 次选方法 | 关键注意 |
|---------|---------|---------|---------|
| **编程开发** | Claude Code + 上下文工程 | Cursor（多文件编辑） | 提供CLAUDE.md，明确技术栈约束 |
| **商业分析** | 结构化提示 + 思维链 | 深度研究Agent | 提供数据上下文，要求量化结论 |
| **内容生产** | 角色设定 + 少样本示例 | 链式提示（大纲→正文→润色） | 给风格样本比描述风格有效10倍 |
| **数据处理** | 代码生成 + 结构化输出 | Agent + 工具调用 | 让AI写代码处理，而非直接处理数据 |
| **学习研究** | 深度研究 + 苏格拉底对话 | RAG + 多轮追问 | 让AI解释给五岁小孩，验证自己理解 |
| **产品设计** | 头脑风暴 + 约束框架 | 多角色辩论 | 先发散再收敛，分两步走 |
| **系统架构** | 推理模型 + 扩展思维 | 分层讨论（先宏观后细节） | 提供完整约束条件，要求权衡分析 |

## 3. 复杂度阶梯

> 原则：从最简单的开始，只在不够用时才升级。

| Level | 方法 | 适用场景 | 典型耗时 |
|-------|------|---------|---------|
| **L0** | 直接提问 | 常识问答、简单翻译、格式转换 | 秒级 |
| **L1** | 精心提示 | 需要特定格式/风格/深度的输出 | 分钟级 |
| **L2** | 少样本 + 上下文 | 有明确样本可参考的任务 | 分钟级 |
| **L3** | 链式提示 | 可分解为固定步骤的多步任务 | 分钟级 |
| **L4** | 单Agent + 工具 | 需要搜索/计算/API调用的任务 | 分钟~小时 |
| **L5** | 多Agent协作 | 需要不同专长协作的复杂任务 | 小时级 |
| **L6** | 多Agent + 人机协同 | 高风险/高价值/需要人类判断的任务 | 天级 |

**升级信号：**
- 输出质量不稳定 → 升一级加约束
- 单次上下文装不下 → 拆分为链式
- 需要外部信息 → 加工具/RAG
- 需要自主判断 → 用Agent
- 单Agent不可靠 → 加监督者/多Agent

## 4. 模型选择指南

### 模型层级

| 层级 | 代表模型 | 适用任务 | 成本 |
|------|---------|---------|------|
| **快速** | Claude Haiku, GPT-4o-mini, Gemini Flash | 分类、提取、简单转换、批量处理 | 极低 |
| **标准** | Claude Sonnet, GPT-4.1, Gemini Pro | 日常编程、写作、分析、对话 | 中等 |
| **推理** | Claude Opus, o3, Gemini Deep Think | 复杂推理、架构设计、数学证明 | 较高 |

### 选择原则

```
1. 先用最好的模型验证任务可行性
2. 可行后，降级测试哪个层级能胜任
3. 生产环境选能胜任的最便宜模型
4. 关键路径用好模型，辅助路径用快模型
```

### 常见搭配模式

| 模式 | 说明 |
|------|------|
| **推理生成 + 快速执行** | Opus写提示词模板 → Haiku批量执行 |
| **快速筛选 + 标准处理** | Haiku做初筛/分类 → Sonnet处理匹配项 |
| **标准主力 + 推理兜底** | Sonnet处理常规 → 低置信度升级到Opus |
| **混合评估** | 快速模型出初稿 → 推理模型评分/改进 |

### 不同厂商优势（2025）

| 厂商 | 核心优势 | 最佳场景 |
|------|---------|---------|
| **Anthropic (Claude)** | 编程、长上下文、指令遵循、安全性 | AI编程、Agent开发、企业应用 |
| **OpenAI (GPT)** | 生态最全、多模态、推理链 | 通用场景、多模态任务 |
| **Google (Gemini)** | 超长上下文（100万token）、搜索集成 | 大文档处理、搜索增强 |
| **DeepSeek** | 性价比极高、中文优秀 | 预算敏感场景、中文任务 |
