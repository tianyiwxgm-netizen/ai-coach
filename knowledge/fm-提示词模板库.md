# 提示词模板库

> 即拿即用的提示词模板。用 `{{变量}}` 标记需要替换的部分。

---

## 一、编程开发场景

### 1.1 代码生成

**场景：** 从需求描述生成功能代码。

```
你是一位{{语言/框架}}高级开发者。

任务：实现以下功能
{{功能描述}}

技术约束：
- 语言/框架：{{技术栈}}
- 需要兼容：{{兼容性要求}}
- 代码风格：{{风格规范，如PEP8}}

要求：
1. 写出生产级代码，包含类型注解和错误处理
2. 关键逻辑添加简明注释
3. 给出使用示例
4. 列出需要注意的边界情况
```

**使用说明：** 技术约束越具体，生成代码越准确。如果有现有代码上下文，一并提供。

**变体：** 加一句"先写伪代码再实现"可提升复杂逻辑的正确率。

---

### 1.2 代码审查

**场景：** 让AI审查代码质量和潜在问题。

```
请审查以下代码，从5个维度评估：

1. **正确性** — 逻辑bug、边界条件、错误处理
2. **安全性** — 注入、泄露、权限问题
3. **性能** — 时间/空间复杂度、不必要的开销
4. **可维护性** — 命名、结构、耦合度
5. **最佳实践** — 是否遵循{{语言/框架}}社区惯例

对每个发现的问题：
- 说明问题和风险等级（高/中/低）
- 给出具体修复建议和修改后代码

代码如下：
```{{语言}}
{{待审查代码}}
```（注意此处反引号是代码块的结束标记）
```

---

### 1.3 调试

**场景：** 让AI帮助定位和修复bug。

```
我遇到了一个bug，需要你帮我定位和修复。

**症状描述：**
{{预期行为 vs 实际行为}}

**错误信息：**
{{错误日志/堆栈跟踪}}

**相关代码：**
{{相关代码片段}}

**已尝试的修复：**
{{已经试过什么}}

请：
1. 分析可能的根因（列出2-3个可能性，按概率排序）
2. 给出验证方法（如何确认是哪个原因）
3. 提供修复方案
```

---

### 1.4 架构设计

**场景：** 系统或模块的架构设计讨论。

```
请帮我设计{{系统/模块名}}的架构。

**业务背景：**
{{这个系统要解决什么问题}}

**功能需求：**
{{核心功能列表}}

**非功能需求：**
- 用户量级：{{预期规模}}
- 性能要求：{{延迟/吞吐量}}
- 可用性要求：{{SLA}}
- 预算约束：{{成本限制}}

**技术约束：**
{{已有技术栈、团队能力、需要集成的系统}}

请输出：
1. 架构总览图（文本描述，说明组件和数据流）
2. 核心组件说明（职责、接口、技术选型）
3. 关键设计决策及权衡分析
4. 潜在风险和应对方案
```

---

### 1.5 测试生成

**场景：** 为已有代码生成测试用例。

```
为以下代码生成全面的测试用例。

测试框架：{{pytest / Jest / JUnit 等}}

```{{语言}}
{{被测代码}}
```（注意此处反引号是代码块的结束标记）

要求覆盖：
1. 正常路径（至少3个典型场景）
2. 边界条件（空值、极值、边界值）
3. 异常路径（无效输入、异常状态）
4. 如果涉及外部依赖，使用mock

每个测试用例命名要清晰表达测试意图。
```

### 1.6 文档撰写

**场景：** 为代码或API生成技术文档。

```
为以下{{代码/API/系统}}撰写技术文档。

{{代码或API定义}}

文档受众：{{新入职开发者 / 外部API使用者 / 内部团队}}

文档包含：
1. 概述（一句话说明这是什么、解决什么问题）
2. 快速开始（最小可用示例）
3. API参考（参数、返回值、异常说明）
4. 使用示例（2-3个常见场景）
5. 注意事项和常见问题
```

## 二、商业分析场景

### 2.1 市场分析

```
请对{{行业/市场}}进行分析。

分析维度：
1. **市场概况** — 规模、增速、发展阶段
2. **竞争格局** — 主要玩家、市场份额、竞争态势
3. **用户画像** — 核心用户群、需求痛点、付费意愿
4. **趋势预判** — 未来2-3年关键趋势、技术驱动因素
5. **机会与风险** — 进入机会点、主要风险

背景信息：{{已知的相关信息}}

输出要求：有数据支撑的结论，标注数据来源的可信度。不确定的地方明确标注"需要验证"。
```

---

### 2.2 竞品分析

```
请对以下竞品进行深度对比分析：

竞品列表：{{竞品A, 竞品B, 竞品C}}
我方产品：{{产品名及定位}}

分析框架：
| 维度 | 竞品A | 竞品B | 竞品C | 我方 |
|------|-------|-------|-------|------|
| 核心功能 | | | | |
| 定价策略 | | | | |
| 目标用户 | | | | |
| 技术优势 | | | | |
| 市场份额 | | | | |
| 融资情况 | | | | |

输出：
1. 对比表格
2. 各竞品SWOT总结
3. 差异化机会点（我们应该在哪里突破）
4. 需要警惕的威胁
```

---

### 2.3 商业方案撰写

```
请帮我撰写{{项目/产品名}}的商业方案。

基本信息：
- 项目背景：{{为什么要做}}
- 目标用户：{{面向谁}}
- 核心价值：{{解决什么问题}}
- 预算范围：{{资金约束}}

方案结构：
1. 执行摘要（一页纸说清楚全部要点）
2. 问题与机会
3. 解决方案
4. 商业模式（怎么赚钱）
5. 市场策略（怎么获客）
6. 财务预测（收入/成本/盈亏）
7. 里程碑与时间表
8. 团队与资源需求
9. 风险与应对

语气：专业但不学术，面向{{投资人/管理层/合作伙伴}}。
```

---

### 2.4 数据分析

```
请分析以下数据并给出洞察。

数据描述：{{数据是什么、怎么来的、时间范围}}

数据内容：
{{粘贴数据或描述数据结构}}

分析目标：{{想回答什么问题}}

请：
1. 数据概览（基本统计量、分布特征）
2. 关键发现（3-5个显著的模式或异常）
3. 原因假设（为什么会出现这些模式）
4. 行动建议（基于分析的具体建议）
5. 如果需要进一步分析，建议补充什么数据
```

## 三、内容生产场景

### 3.1 长文撰写

```
请撰写一篇关于{{主题}}的文章。

基本参数：
- 字数：{{字数范围}}
- 受众：{{目标读者}}
- 发布渠道：{{微信公众号 / 博客 / 知乎 等}}
- 语气风格：{{专业严谨 / 轻松有趣 / 深度分析}}

文章要求：
- 开头要有吸引力（不要用"随着...的发展"开头）
- 每个论点要有具体案例或数据支撑
- 结尾要有行动号召或思考启发
- {{附加要求}}

参考风格：
{{粘贴一段你喜欢的风格样本}}
```

**变体：** 先让AI输出大纲，确认后再写正文，效果更好。

---

### 3.2 营销文案

```
为{{产品/活动}}撰写营销文案。

产品信息：
- 产品名：{{名称}}
- 核心卖点：{{1-3个}}
- 目标用户：{{画像}}
- 用户痛点：{{解决什么问题}}

文案要求：
- 渠道：{{朋友圈 / 小红书 / 抖音 / 落地页}}
- 字数限制：{{字数}}
- 行动号召：{{希望用户做什么}}

请提供3个版本：
1. 理性版（强调功能和数据）
2. 感性版（强调情感和场景）
3. 紧迫版（强调限时/限量/机会）
```

### 3.3 技术博客

```
请撰写一篇技术博客，主题：{{主题}}

读者定位：{{初学者 / 中级开发者 / 高级开发者}}

文章结构：
1. 问题引入（为什么需要关注这个话题）
2. 核心概念解释
3. 实战演示（完整可运行的代码示例）
4. 踩坑记录（常见错误和解决方案）
5. 最佳实践总结
6. 延伸阅读推荐

风格要求：
- 代码示例要完整可运行
- 用类比解释抽象概念
- 加入个人经验和观点
```

视频信息：

- 结尾有互动引导（点赞/关注/评论）
```

## 四、学习研究场景

### 4.1 深度研究

```
请对{{研究主题}}进行深度研究。

研究范围：
- 核心问题：{{要回答什么问题}}
- 时间范围：{{关注哪个时期}}
- 地域范围：{{全球 / 特定地区}}

输出结构：
1. 研究摘要（300字以内）
2. 背景与现状
3. 主要发现（分3-5个主题组织）
4. 不同观点与争议
5. 未来趋势预判
6. 参考信息来源

要求：区分"已验证的事实"和"推测性分析"。标注信息的可信度级别。
```

---

### 4.2 概念学习

```
请帮我彻底理解{{概念/技术名称}}。

我的背景：{{当前知识水平，已经知道什么}}

请按以下顺序讲解：
1. 一句话定义（小学生能听懂）
2. 日常类比（用熟悉的事物类比）
3. 正式定义（准确的技术描述）
4. 核心原理（为什么这样设计/工作）
5. 实际例子（2-3个真实应用场景）
6. 常见误区（大多数人会搞错什么）
7. 和相关概念的对比（{{相关概念A}} vs {{相关概念B}}）
8. 自测问题（3个问题检验我是否真的理解了）
```

---

### 4.3 论文分析

```
请分析以下论文：

{{论文标题/摘要/链接/全文}}

分析维度：
1. **核心贡献** — 这篇论文解决了什么问题？方法的新颖性在哪？
2. **方法概述** — 用简明语言解释技术方法（不用公式）
3. **实验结果** — 关键指标和对比基线
4. **优势与局限** — 方法的优点和不足
5. **实践价值** — 对工程实践有什么启发？能直接用吗？
6. **相关工作** — 和哪些其他工作有关联？

输出面向：{{研究者 / 工程师 / 管理者}}
```

## 五、通用场景

### 5.1 头脑风暴

```
请围绕{{主题/问题}}进行头脑风暴。

背景：{{当前情况和约束}}

规则：
1. 先发散：生成20个想法，不做任何评判，越wild越好
2. 再分类：按可行性分为"立即可做"、"需要条件"、"异想天开"三类
3. 再聚焦：从"立即可做"中选出最有潜力的3个，每个展开说明
4. 最后：给出一个你认为最值得尝试的方案和理由
```

---

### 5.2 决策支持

```
我需要在以下选项中做出决策：

选项列表：
A. {{选项A}}
B. {{选项B}}
C. {{选项C}}

决策背景：{{当前情况}}
关键约束：{{时间/预算/资源/风险偏好}}
决策目标：{{优化什么}}

请：
1. 每个选项的优劣分析（至少各列3点）
2. 按{{评估维度1, 维度2, 维度3}}打分（1-10分）
3. 风险评估（最坏情况是什么）
4. 推荐方案和理由
5. 如果推荐方案失败，Plan B是什么
```

---

### 5.3 总结提炼

```
请总结以下内容：

{{原始内容（文章/会议记录/报告/对话）}}

总结要求：
- 格式：{{一句话摘要 / 要点列表 / 结构化摘要}}
- 长度：{{原文的1/10 / 300字以内 / 5个要点}}
- 侧重点：{{关键结论 / 行动项 / 争议点}}
- 受众：{{汇报给老板 / 团队同步 / 个人备忘}}

输出结构：
1. 一句话核心结论
2. 关键要点（3-5点）
3. 行动项（如有）
4. 遗留问题（如有）
```
