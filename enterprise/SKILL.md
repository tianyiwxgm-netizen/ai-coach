# 企业级/巨型级项目知识库

本知识库由 ai-coach 主 skill 在诊断出企业级或巨型级复杂度时调用。角色专项内容已分别移入 product/、backend/ 等子 skill。

---

## 巨型项目全链路六阶段框架

**适用场景：** 淘宝级系统重构、企业数字化转型、多团队大型项目

```
阶段 A：战略分析（Why）
  ↓ 产出：项目愿景 + 成功标准 + 核心假设
阶段 B：业务架构（What - 业务视角）
  ↓ 产出：业务模式画布 + 核心流程 + 利益相关者图
阶段 C：产品设计（What - 产品视角）
  ↓ 产出：PRD + 用户故事地图 + 产品路线图
阶段 D：技术架构（How - 架构视角）
  ↓ 产出：技术方案 + 系统拓扑 + 数据架构
阶段 E：实施规划（How - 执行视角）
  ↓ 产出：分阶段路线图 + 风险评估 + 团队分工
阶段 F：迭代执行（Do）
  ↓ 产出：每阶段交付物 + 复盘报告
```

### 每阶段推荐 Skill 组合

| 阶段 | 推荐 Skill | 产出物 |
|------|-----------|--------|
| A 战略分析 | `/brainstorm` + `/deep-think` | 战略愿景文档 |
| B 业务架构 | `/brainstorm` + `/jobs-to-be-done` | 业务模式画布 |
| C 产品设计 | `/prd-development` + `/user-story-mapping` + `/user-story` | PRD + 用户故事 |
| D 技术架构 | `/brainstorm` + `/deep-think` | 技术方案文档 |
| E 实施规划 | `/write-plan` | 分阶段路线图 |
| F 迭代执行 | `/execute-plan` + `/debug` | 代码 + 测试 |

### 巨型项目特殊要求

1. **分阶段产出** — 不一次性产出全部，每个阶段产出一份引导指南
2. **阶段间交付物传递** — 上一阶段的产出是下一阶段的输入，引导指南中明确说明
3. **阶段验收点** — 每个阶段完成后，建议用 `/ai-coach` 复盘，确认方向正确再进入下一阶段
4. **风险持续追踪** — 每个阶段的引导指南都包含风险更新

## 企业级/巨型级通用知识库

### 跨团队协调要点

- 识别跨团队依赖：哪些接口/数据/服务跨团队边界
- 治具工程思维：共享 CLAUDE.md、约束系统、一致性保障
- 沟通机制：接口契约、变更通知流程、联调计划

### 架构分析框架

- 系统全景：服务拓扑、数据流向、关键路径
- 技术债评估：阻塞性债务（必须先还）vs 可共存债务
- 风险热力图：影响面 x 发生概率排序

### 风险评估模板

| 风险项 | 影响面 | 概率 | 缓解措施 | 回退方案 |
|-------|-------|------|---------|---------|
| {描述} | 高/中/低 | 高/中/低 | {措施} | {回退} |

### 分阶段路线图模板

```
# {项目名} 路线图

## 阶段 1：{标题}（预计 {N} 周）
- 目标：{具体目标}
- 涉及范围：{仓库/模块/团队}
- 验收标准：{可验证的标准}
- 风险点：{风险描述}
- 回退方案：{回退策略}

## 阶段 2：...
```

### 渐进替换策略（Strangler Fig）

新旧系统并行，逐步切流量到新系统。适用于不能停机的生产系统重构。

### 评估驱动开发

先建评估体系再动手 — 定义成功指标、建立基线、每阶段对比验证。
