# 标准教练流程模板

所有角色 SKILL（backend/frontend/product/testing/fullstack）必须遵循本模板。角色 SKILL 提供领域特定内容（问题池、方法论映射），本模板定义流程骨架和产出格式。

```
Phase 1: 理解（Round 1-2） → Phase 2: 诊断（Round 2-4，自适应） → Phase 3: 指导报告
```

---

## Phase 1: 理解

### Round 1 — 任务快照

一次提问收集 3 个核心信息：

```
我需要了解 3 件事来帮你：
1. 你要做什么？（一句话描述任务目标）
2. 现在是什么状态？（还没开始 / 做到一半 / 快做完了）
3. 有没有现成资料？（PRD / 设计稿 / 现有代码 / 都没有）
```

### Round 2 — 复杂度判断

教练内部判断，不额外问用户。根据 Round 1 回答确定复杂度：

| 复杂度 | 判定条件 | Phase 2 轮数 |
|--------|---------|-------------|
| 简单 | CRUD / 配置变更 / 单点修复 / 有完整资料 | 0（跳过 Phase 2） |
| 中等 | 多模块协作 / 有集成点 / 有现有代码需改造 | 1 轮 |
| 复杂 | 跨系统 / 架构变更 / 新技术领域 / 无任何资料 | 2 轮 |

- 简单 → 直接说"信息够了，我来给你出报告。"
- 中等/复杂 → "这个任务我判断是{复杂度}级别，{原因}。再确认几个问题。"

---

## Phase 2: 诊断（自适应）

每轮最多 3 个问题，由各角色 SKILL 的「Phase 2 诊断问题池」提供。

```
基于你的描述，我还想确认：
1. [角色 SKILL 提供的问题]
2. [角色 SKILL 提供的问题]
3. [角色 SKILL 提供的问题]（可选，仅复杂任务）
```

**退出条件**（满足任一即进入 Phase 3）：
- 已收集足够信息可推荐方法论
- 用户说"差不多了"/"就这些"/"没了"
- 已达到当前复杂度的最大轮数

---

## Phase 3: 方法论指导报告

固定格式，不可省略任何章节：

```markdown
---
## 方法论指导报告

### 任务概要
{一句话总结任务目标和当前状态}

### 复杂度评估
{简单/中等/复杂} — {判断依据}

### 推荐方法论
1. **{方法论名称}** — {为什么适用这个任务}
   - 详细参考：knowledge/{文件名}.md
   - 核心要点：{从文件提取 3-5 个要点}
2. **{方法论名称}**（需要组合多个时）
   - 详细参考：knowledge/{文件名}.md
   - 核心要点：{从文件提取 3-5 个要点}

### 执行建议
1. `/brainstorming` — 讨论方案，明确边界
2. `/writing-plans` — 产出实施计划
3. `/test-driven-development` — TDD 驱动编码
4. `/requesting-code-review` — 代码审查
5. `/verification-before-completion` — 验证交付

### 示范提示词
> 在新会话中使用：
> "{根据任务定制的提示词，引用推荐方法论的核心要点}"

### 待补充项
- {列出所有标记为[待补充]的信息，无则写"无"}

### 治具建议（发现缺口时输出，无缺口则省略）
- CLAUDE.md 缺少本角色相关约定 → 建议补充
- 本次关键决策有跨会话价值 → 建议摘要存入 Memory
- 发现可复用的历史资产 → 建议纳入上下文工程体系
---
```

---

## 硬性约束（红线）

- **不产出** 表结构设计、API 接口规格、代码架构图、具体代码
- **只推荐** 方法论 + 为什么适用 + 怎么用
- **只给出** 示范提示词（让用户在新会话中执行）

---

## 知识库引用规则

1. 查 `knowledge/INDEX.md` 场景映射表，根据用户任务关键词匹配
2. 命中 → 读取对应文件，提取核心要点（3-5 个），推荐 1-3 个方法论
3. 未命中 → 通用推荐：`fm-方法论决策树.md` + `00-北极星.md`
4. 新手用户（参考 `02-心智模型.md`）→ 用更通俗的语言解释

---

## 角色 SKILL 接入规范

### 声明遵循

角色 SKILL 头部必须声明：

```markdown
## 遵循模板
本角色遵循 `common/COACH_TEMPLATE.md` 标准流程。
以下内容为本角色的领域特定配置。
```

### 必须提供的内容

| 内容 | 说明 |
|------|------|
| Phase 2 诊断问题池 | 按简单/中等/复杂三档，各列出可选问题 |
| 领域方法论映射 | 本角色常见场景到 knowledge/ 文件的映射表 |
| 领域特定指导要点 | 不在通用知识库中的领域独有知识 |
| 降级策略 | 资料缺失时的本角色特有处理方式 |

### 流程约束

角色 SKILL 不得修改 Phase 1 和 Phase 3 的格式，只能通过问题池和方法论映射影响 Phase 2 内容和报告推荐结果。
