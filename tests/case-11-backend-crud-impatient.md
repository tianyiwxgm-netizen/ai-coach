# Case 11: 后端 CRUD - 用户嫌问太多

## 基本信息

- **分类**: 后端角色 / 用户体验 / 信息收集流程
- **优先级**: P0
- **可能翻车的点**: B2 要收集 7 项信息（PRD、原型、接口规范等），但用户明确表达"别问那么多"。全局规则 8 的"快速提问模式"只在主菜单触发，角色引导流程中没有"简化收集"的机制。教练可能无视用户意愿，继续逐项询问 7 个问题。

## 真实用户输入

```
/ai-coach "做个供应商的增删改查，别问那么多直接告诉我怎么开始"
```

## SKILL.md 应该怎么处理

1. 识别到用户意图是"后端新功能"，路由到 B2 上下文收集
2. 识别到用户明确表达了"别问那么多"的情绪 — 不应该逐项问 7 个问题
3. 将必要信息合并为 1-2 个问题（而非 7 个），或直接基于"供应商增删改查"这个足够明确的描述给出起步方案
4. 快速进入 B3 方案设计，给出 CRUD 脚手架方案

## 验收标准

- [x] AC1: 正确识别为后端新功能任务，进入后端角色流程
- [ ] AC2: 不逐项询问 7 个信息收集问题（最多合并为 1-2 个关键问题）
- [ ] AC3: 尊重用户"别问那么多"的意愿，不出现"我需要先了解以下几个方面"之类的长清单
- [ ] AC4: 在第一轮或第二轮交互内给出可操作的起步方案（如表结构、Controller 骨架）
- [x] AC5: 仍然传达 TDD 铁律（但不应因此增加更多提问轮次）
- [ ] AC6: 没有出现教练式说教语气（如"为了帮你做好规划，我需要..."）
