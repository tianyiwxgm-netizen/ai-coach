# AI-Coach v3 测试执行计划

## 概述

本计划覆盖 ai-coach skill 的全部 **71 个测试用例**，分两大阶段：

- **v3 新增用例（case-51 ~ case-71）**：21 个，验证 v3 五角色子 skill 架构
- **v2 历史用例（case-01 ~ case-50）**：50 个，验证 v2 单 skill 架构（已执行，结果见文末）

每个用例的执行流程：
1. 读取用例文档，理解场景和预期行为
2. 模拟用户输入，通过 Skill 工具调用 ai-coach / ai-coach-{角色}
3. 按照用例描述与 ai-coach 交互
4. 记录完整对话过程
5. 对照验收标准逐项检查
6. 将执行记录、对话记录、测试结果回写到用例文档

## 执行方法

### 单用例执行流程

```
1. 读取 case-NN-xxx.md，理解场景
2. 通过 Skill 工具调用 ai-coach（或 ai-coach-{角色}）
3. 按照用例中的"用户输入"模拟交互
4. 记录 ai-coach 的每一步响应
5. 对照"预期行为"检查是否匹配
6. 逐项检查"验收标准"，标记 ✅ 或 ❌
7. 将以下内容回写到用例文档：
   - 执行记录：执行时间、环境、执行者
   - 对话记录：完整的交互内容
   - 测试结果：PASS/FAIL + 各验收标准检查结果 + 发现的问题
```

### 结果判定

| 结果 | 判定标准 |
|------|---------|
| PASS | 所有验收标准通过 |
| PARTIAL | 超过 50% 验收标准通过，但有部分不满足 |
| FAIL | 超过 50% 验收标准未通过，或存在阻塞性问题 |

### 问题分类

| 类型 | 说明 | 处理方式 |
|------|------|---------|
| BUG | ai-coach 行为与 SKILL.md 定义不一致 | 修复 SKILL.md |
| ENHANCEMENT | 行为可接受但可以改进 | 记录为改进建议 |
| DESIGN | SKILL.md 设计本身需要调整 | 提出设计变更建议 |
