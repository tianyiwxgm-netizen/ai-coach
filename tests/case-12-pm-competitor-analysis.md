# Case 12: 产品经理 - 竞品分析独立报告

## 基本信息

- **分类**: 产品角色 / 流程边界 / 任务类型不匹配
- **优先级**: P1
- **可能翻车的点**: 产品角色 P1 快捷路由表有"竞品分析->P3.3 竞品分析"，但 P3.3 是 PRD 流程的子步骤（PRD 的竞品分析章节），不是独立竞品报告产出。用户要的是一份独立的竞品分析报告，不是嵌在 PRD 里的竞品分析段落。教练可能强行把用户拉进 PRD 全流程（P1->P2->P3），用户只想要 P3.3 的产出。

## 真实用户输入

```
/ai-coach "帮我分析一下美团和饿了么的会员体系，做个竞品报告"
```

## SKILL.md 应该怎么处理

1. 识别到关键词"竞品分析"，路由到 P3.3
2. 判断用户的真实需求：独立竞品报告 vs PRD 中的竞品章节
3. 如果 P3.3 设计为 PRD 子步骤，应该能独立抽出来用，或告知用户这个限制
4. 引导用户提供竞品分析所需的维度、对比范围，产出独立的竞品分析报告

## 验收标准

- [x] AC1: 正确识别为"竞品分析"任务，不误判为"写 PRD"
- [x] AC2: 不强制要求用户走完 P1（历史资产盘点）、P2（需求分析）才能做竞品分析
- [ ] AC3: 产出物是独立的竞品分析报告，不是 PRD 文档中的一个章节
- [ ] AC4: 引导用户明确对比维度（如会员等级、权益、价格、留存策略等）
- [ ] AC5: 报告结构清晰，包含对比表格或矩阵，有可操作的结论
- [x] AC6: 不出现"我们先完成 PRD 的前置步骤"之类的流程强制

### 发现的 SKILL.md 缺陷

1. **P3.3 竞品分析缺乏独立报告产出能力**：P3.3 设计为 PRD 流程的子步骤，产出的竞品对比表是 PRD 的一个章节。当用户单独需要完整竞品分析报告时，缺少独立的报告模板（应包含：分析背景/对比维度/竞品概况/对比矩阵/结论与建议/行动计划）。
2. **P3.3 对比维度固定化**：5 个固定维度（核心功能/用户体验/定价策略/技术实现/差异化优势）不够灵活。应支持根据用户需求自定义维度，或根据分析对象自动推荐维度。
3. **P3.3 缺少结论与行动建议段**：竞品分析的价值在于"so what"——对比完之后怎么做。当前 P3.3 只有对比表格和一个引导提问，没有要求产出结论和行动建议。
