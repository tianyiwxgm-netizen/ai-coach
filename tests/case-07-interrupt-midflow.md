# Case 07: 引导过程中突然问无关技术问题

## 基本信息

- **角色**: 后端开发
- **场景**: 教练正在 B2 收集技术上下文（已收集到第 3 项），用户突然想到一个跟当前任务无关的技术问题
- **前置条件**: Memory 中已有 `AI教练角色: 后端`，正在执行 B2 收集流程

## 真实用户输入

对话上下文：
```
教练：你好！我是后端教练。你这次要做什么？
用户：帮我做个新功能，商品分类管理
教练：好的，我来帮你准备。PRD 文档在哪？
用户：在 docs/plans/ 下面
教练：收到。有设计稿或原型吗？
用户：没有
教练：好的，标记[待补充]。这个需求涉及哪些模块？
```

然后用户突然：
```
对了，Spring Boot 3.5 支持虚拟线程吗？我看到个文章说可以用
```

## SKILL.md 应该如何处理

1. 当前状态：B2 收集第 3 项"现有代码结构"
2. 用户输入与当前收集流程无关
3. 全局规则第 4 条"中途切换"：任何步骤可切换角色或任务类型
4. 但这不是切换任务——这是临时插入的无关问题

## 验收标准

- [ ] 不应该把"Spring Boot 3.5 虚拟线程"当成 B2 第 3 项"现有代码结构"的回答
- [ ] 应该识别出这是一个离题的插入问题
- [ ] 可以简短回答这个技术问题（全局规则第 8 条"快速提问"的精神），然后拉回 B2 流程
- [ ] 回到 B2 时，之前收集的信息（PRD 路径、无设计稿）不应该丢失
- [ ] 不应该建议"开新会话"来问这个小问题（过度正式）

## 可能翻车的点

1. **全局规则没有覆盖"临时插入问题"场景**：第 4 条"中途切换"处理的是角色或任务切换（比如"我不做这个了，换个任务"），不是临时提问。第 8 条"快速提问模式"需要在主菜单选"快速提问"才触发。在 B2 收集过程中的插入问题，两条规则都不完全适用
2. **教练可能把无关回答当作收集输入**：如果 SKILL.md 的实现是按序收集，期待用户每轮回答当前收集项，那"Spring Boot 3.5 支持虚拟线程吗"可能被当成第 3 项的回答，导致数据污染
3. **全局规则第 1 条的边界**：教练不执行业务任务。回答"Spring Boot 3.5 是否支持虚拟线程"算不算业务任务？这是一个通用技术知识问题，应该可以简单回答，但 SKILL.md 没有明确允许
4. **上下文恢复**：如果教练回答了虚拟线程的问题，然后说"好的，回到刚才的话题"，Claude 的上下文窗口是否能准确记住 B2 收集到了第几项？如果教练回答比较长（解释虚拟线程的用法），可能会稀释之前的收集上下文
5. **用户可能连续插入**：如果用户发现教练会回答无关问题，可能接着问更多无关问题，教练需要有策略应对
