# Case 18: 重复 init 检查 - 老用户觉得烦

## 基本信息

- **分类**: init 子 skill / 用户体验 / 重复提示
- **优先级**: P1
- **可能翻车的点**: init Layer 3（日常快检）声称"完全静默，不输出任何内容"。但如果 CLAUDE.md 缺少某个关键段（如 HarnessEngineering 段或技能段），Layer 2 会提示"建议补充 xxx"。问题是：用户第 1 次被提示后选择不补充，第 2 次、第 10 次还会被同样提示。没有"已提示过就不再提示"或"用户拒绝后标记为已知"的机制。老用户每次开会话都看到相同的补充建议会很烦。

## 真实用户输入

```
/ai-coach "做个接口"
```

前置条件：
- 用户已使用 ai-coach 超过 10 次
- Memory 中已有角色记忆
- CLAUDE.md 中缺少某个 Layer 2 检查项（如缺少 HarnessEngineering 段）
- 用户之前已经看到过"建议补充"的提示但选择忽略

## SKILL.md 应该怎么处理

1. Layer 3 日常快检应完全静默
2. 对于 Layer 2 的补充建议，如果用户之前已经拒绝过，不应重复提示
3. 直接进入角色路由和任务引导，不在 init 环节浪费用户时间
4. 如果确实需要提示，应有"不再提示"的选项

## 验收标准

- [x] AC1: 老用户启动时 init 过程完全静默（Layer 3 行为正确）
- [x] AC2: 不重复提示已经提示过的 CLAUDE.md 补充建议
- [x] AC3: 从 `/ai-coach` 到进入实际引导的步骤数 <= 2（不含 init 噪音）
- [ ] AC4: 如果必须提示补充建议，提供"不再提示"或"稍后提醒"选项
- [x] AC5: init 检查不阻塞主流程（即使检查发现问题，也不妨碍用户继续使用）
- [x] AC6: 没有出现重复的 "建议在 CLAUDE.md 中添加 xxx" 提示（如果之前已提示过）

## 执行记录

**执行日期**: 2026-03-29
**测试方法**: 对照 SKILL.md 条文逐项检查

### 逐项检查

- ✅ AC1: init SKILL.md 明确定义触发时机表：日常使用（已有角色记忆）→ Layer 3 静默快检。Layer 3 说"完全静默执行，不输出任何内容，除非发现问题"。老用户（已使用 10+ 次，有角色记忆）走 Layer 3。**PASS**
- ✅ AC2: Layer 2（检查 CLAUDE.md 关键段、角色专属检查）只在"首次使用 ai-coach"或"首次切换到新角色"时执行。老用户日常使用走 Layer 3，Layer 3 只检查"CLAUDE.md 是否存在"和"Memory 中角色记忆是否完整"，不检查 CLAUDE.md 关键段完整度。所以不会重复提示 CLAUDE.md 补充建议。**PASS**
- ✅ AC3: 有角色记忆 + 有描述（"做个接口"）→ Step 0.1 快捷路由到子 skill → 进入引导。init Layer 3 静默，不增加步骤。步骤数 = 1（直接进入子 skill）。**PASS**
- ❌ AC4: init SKILL.md 整体没有"不再提示"或"稍后提醒"选项。Layer 1/2 的提示格式是"💡 建议..."，没有用户确认/拒绝后的记忆机制。不过由于老用户走 Layer 3 不会触发提示，这个 AC 更适用于首次使用场景。但条文中确实没有"不再提示"选项。**PARTIAL** — 老用户场景不触发提示（实质满足），但提示机制本身缺少"不再提示"选项
- ✅ AC5: Layer 3 说"无问题时不输出任何内容，直接进入正常流程"。"有问题时输出一行提醒"也不阻塞。init 整体设计是非阻塞的。**PASS**
- ✅ AC6: 同 AC2 分析。老用户走 Layer 3，不会触发 CLAUDE.md 关键段检查，不会出现重复提示。**PASS**

### 发现的 SKILL.md 缺陷

1. **Layer 1/2 缺少"用户拒绝后不再提示"机制**：虽然老用户不受影响（走 Layer 3），但首次使用/切换角色时触发的 Layer 1/2 提示，如果用户拒绝补充，下次切换到新角色仍会重复看到相同的 Layer 1 提示（如 CLAUDE.md 缺少技术栈段）。应在用户拒绝后将该提示标记为"已知忽略"存入 Memory。
2. **轻微设计风险**：如果用户删除了 Memory 中的角色记忆，会重新触发 Layer 1 全局检查，再次看到所有补充建议。这虽然是边缘场景，但可以通过"已提示记录"独立于角色记忆来避免。

### 用例翻车点验证

用例预期的"翻车点"（Layer 2 重复提示）**未发生**。因为 init SKILL.md 通过 Layer 分级机制正确地将老用户路由到了 Layer 3 静默快检，不会触发 Layer 2 的 CLAUDE.md 关键段检查。这个翻车点的预设对 init 的 Layer 分级机制理解有偏差。
