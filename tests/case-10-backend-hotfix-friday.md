# Case 10: 周五下午线上故障，开发需要紧急修复

## 基本信息

- **角色**: 后端开发
- **场景**: 周五下午 5 点，线上订单接口突然返回 500，影响正常运营。开发需要赶紧修复
- **前置条件**: Memory 中已有 `AI教练角色: 后端`

## 真实用户输入

```
线上挂了！订单接口 500 了，赶紧帮我
```

## SKILL.md 应该如何处理

1. Memory 有角色记忆（后端）+ 带任务描述 → 快捷路由
2. B1 快捷路由表：
   - "Bug / 报错 / 不工作 / 异常 / 失败" → B-Bug
   - "紧急 / 线上故障 / 生产事故 / 赶紧" → B-Emergency
3. 用户输入中"赶紧"匹配 B-Emergency，"挂了"和"500"也暗示紧急
4. → 路由到 B-Emergency 流程
5. B-Emergency Phase 1：快速定位（5 分钟内）→ Phase 2：止血方案 → Phase 3：事后补充

## 验收标准

- [ ] "赶紧"关键词应成功路由到 B-Emergency 而不是 B-Bug
- [ ] 不应该在紧急情况下问一堆信息收集问题（B-Bug 的 5 项信息收集）
- [ ] B-Emergency 的引导应该直入主题：看日志、判断影响、选止血方案
- [ ] 引导方式应该是指令式（"先做这个，再做那个"），不是讨论式
- [ ] 不应该在紧急修复时推荐"开新会话用 /systematic-debugging"——时间不允许
- [ ] 事后补充（Phase 3）可以在修复完成后再引导，不要在紧急阶段就提

## 可能翻车的点

1. **路由关键词匹配的精确度**：用户说"挂了"——这个词不在 B-Emergency 的关键词列表中（紧急 / 线上故障 / 生产事故 / 赶紧）。"赶紧"在列表中，可以匹配。但如果用户只说"线上挂了！订单接口 500 了"不带"赶紧"，"挂了"不在 Emergency 列表，"500"不在 Bug 列表。路由可能失败，落到主菜单
2. **教练不执行业务任务 vs 紧急场景**：全局规则第 1 条说教练不执行业务任务。但紧急场景下用户说"赶紧帮我"，期望教练直接帮忙查日志、定位问题。教练只能"引导"（给示范提示词让用户开新会话执行），这在紧急场景下是致命的时间浪费
3. **B-Emergency 的流程仍然偏慢**：Phase 1 "快速定位（5 分钟内）"列了 3 个步骤。Phase 2 止血方案是一个表格让你选。这些步骤本身是合理的，但教练的输出方式（生成引导文档+示范提示词）跟紧急场景需要的速度不匹配
4. **"开新会话"建议在紧急场景是灾难性的**：全局规则第 2 条说"引导结束后建议用户开新会话执行"。在紧急场景下建议开新会话意味着：离开当前对话 → 新建会话 → 复制提示词 → 等待加载 → 开始执行。每一步都浪费时间
5. **Phase 3 事后补充的时机**：SKILL.md 把 Phase 3（补测试+复盘）放在 B-Emergency 流程里，但没有说"等紧急修复完成后再进入 Phase 3"。如果教练在 Phase 2 止血完成后立刻引导做 Phase 3，用户可能刚松口气就被要求写复现测试，体验很差
6. **止血方案的决策支持不够**：Phase 2 给了两个方案（回滚 vs 定点修复），但怎么判断"该回滚还是定点修复"？新手可能不知道怎么选。SKILL.md 给了"适用场景"的简单描述，但实际决策需要更多信息（比如：上次部署是什么时候？最近改了什么代码？有没有回滚脚本？）
7. **与 case-03 的边界**：case-03 的"线上报错了"被路由到 B-Bug，这里的"线上挂了"被路由到 B-Emergency。两者的区分只靠"赶紧"这个词，但实际严重程度可能差不多

### 发现的 SKILL.md 缺陷

1. **B-Emergency 缺少语气/风格指引**：紧急场景下教练应该用指令式语气（"立即执行"），而非引导式语气（"你觉得要不要"）。SKILL.md 没有规定 B-Emergency 的输出语气。
2. **全局规则第 1 条和第 2 条在紧急场景下的冲突**：教练不执行业务任务（只给提示词）+ 建议开新会话执行 = 紧急场景下至少增加 2-3 分钟延迟（新建会话+复制提示词+等待加载）。SKILL.md 应该为 B-Emergency 增加豁免条款：紧急场景下教练可以直接协助执行诊断命令（如查看日志），而非只给提示词。
3. **止血方案缺少决策指引**：Phase 2 给了两个方案（回滚 vs 定点修复），但没有帮用户做决策的引导问题（如"上次部署是什么时候？""最近改了什么？""有没有回滚脚本？"）。新手可能不知道怎么选。
4. **"挂了"不在 B-Emergency 关键词列表中**：如果用户只说"线上挂了！订单接口 500 了"不带"赶紧"，"挂了"不在 Emergency 列表中，"500"也不在 Bug 列表中。路由可能失败。关键词覆盖度不够。
