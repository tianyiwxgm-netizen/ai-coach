# Case 13: 测试工程师 - 纯技术知识问题

## 基本信息

- **分类**: 测试角色 / 快速提问 / 任务误判
- **优先级**: P1
- **可能翻车的点**: 用户问的是 Playwright 技术知识，不是要做测试规划。"快速提问"模式的触发条件是主菜单选择（选项 8），不是自动检测问题类型。如果用户已有测试角色记忆且直接带问题进入，Step 0.1 会检测到有角色记忆 + 有"任务描述"，可能把"Playwright 的 locator 怎么写"当成测试任务，走 T1->T2 流程开始收集被测系统信息。

## 真实用户输入

```
/ai-coach "Playwright 的 locator 怎么写？有几种选择器？"
```

前置条件：Memory 中已有 `AI教练角色: 测试`

## SKILL.md 应该怎么处理

1. 识别到这是一个技术知识问题（"怎么写"、"有几种"），不是测试任务
2. 不走 T1（被测系统确认）-> T2（测试范围分析）流程
3. 直接回答 Playwright locator 的用法和选择器类型
4. 回答完毕后，询问是否还有其他问题或需要开始测试任务

## 验收标准

- [ ] AC1: 不启动测试规划流程（T1/T2），不询问"被测系统是什么"
- [ ] AC2: 直接回答 Playwright locator 的技术知识（getByRole, getByText, css, xpath 等）
- [ ] AC3: 回答内容技术准确，包含代码示例
- [ ] AC4: 回答篇幅适中，不展开到测试策略、测试计划等无关内容
- [ ] AC5: 回答后自然收尾，不强制引导进入测试规划流程
- [ ] AC6: 不出现"在开始之前，我需要了解你的项目"之类的引导话术
