# Case 19: 产品经理 - PRD 写完后不知道怎么交接

## 基本信息

- **分类**: 产品角色 / 交接流程 / 新手理解力
- **优先级**: P1
- **可能翻车的点**: P6 交付物检查+交接指引说"新会话使用 /ai-coach 后端角色"。但 Claude Code 新手可能不理解"开新会话"是什么操作（关掉重开？换个终端？），也不清楚后端同事是否已安装 ai-coach、是否有自己的 CLAUDE.md、怎么接收 PRD 文件。交接指引可能太抽象，缺少具体的操作步骤。

## 真实用户输入

第一步（PRD 流程完成后）：
```
PRD 写完了，然后呢？怎么让后端开始做？
```

## SKILL.md 应该怎么处理

1. 确认 PRD 交付物完整性（文件在哪、包含什么）
2. 给出具体的交接操作步骤，而非抽象的"开新会话"
3. 说明后端同事需要什么前置条件（安装 ai-coach、有 CLAUDE.md 等）
4. 提供可复制的交接文本/提示词，产品经理可以直接发给后端同事
5. 如果后端同事没有 ai-coach，给出替代方案

## 验收标准

- [ ] AC1: 告知用户 PRD 产出物的具体位置（文件路径）
- [ ] AC2: 交接步骤具体可操作（不是"开新会话用后端角色"这种抽象描述）
- [ ] AC3: 包含给后端同事的交接信息模板（可复制粘贴发给同事）
- [ ] AC4: 说明后端同事的前置条件（需要安装什么、配置什么）
- [ ] AC5: 考虑后端同事没有 ai-coach 的情况，给出降级方案
- [ ] AC6: 不假设用户了解 Claude Code 的"会话"概念
- [ ] AC7: 交接指引中包含 PRD 的关键信息摘要（不是让后端自己去读整个 PRD）
