Phase 3 · Round 3 / 3 — 输出报告

🎯 方法论指导报告

任务概要

为现有 JWT+Spring Security 系统新增 RBAC 权限管理（API 级粒度）

复杂度评估

中等 多表设计 + 中间件集成 + 现有系统改造，但 PRD 已有、粒度明确

推荐方法论

1. 上下文工程 — 你是第一次做权限，需要先让 AI 充分理解你的现有系统

  • 📖 knowledge/04-上下文工程.md
  • 🔑 核心要点：
    • 先把现有用户表结构、Security 配置、登录流程喂给 AI
    • PRD 内容作为约束条件一起提供
    • 控制信噪比：只给权限相关的代码，不要全项目
    • 让 AI "看到"你的技术栈约束（Spring Boot 版本、数据库类型）

2. 治具工程 — 在 CLAUDE.md 中固化权限设计约定，避免 AI 每次重新理解

  • 📖 knowledge/06-治具工程.md
  • 🔑 核心要点：
    • 完成权限设计后，把 RBAC 表结构和 API 约定写入 CLAUDE.md
    • 后续所有权限相关开发，AI 自动遵循已有约定
    • 一次性投入 > 每次重复说明

3. 评估驱动开发 — 先定义"权限系统怎样算合格"，再开始编码

  • 📖 knowledge/07-评估驱动开发.md
  • 🔑 核心要点：
    • 先写权限校验的测试用例（谁能访问什么、谁不能访问什么）
    • 用测试驱动表结构和 API 设计，而非凭感觉设计
    • TDD 流程：Red → Green → Refactor

执行建议

开新会话，按以下顺序执行：

  1. /brainstorming — 讨论 RBAC 表设计方案（带上 PRD + 现有代码）
  2. /writing-plans — 拆分实施步骤
  3. /test-driven-development — 先写权限测试，再写实现
  4. /requesting-code-review — 完成后审查
  5. /verification-before-completion — 最终验证

设计完成后把约定写入 CLAUDE.md。"
